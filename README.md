# Автоматизированный анализатор аншлифов руд (Тальк и сульфидные срастания)

Программное решение на базе искусственного интеллекта и математической морфологии для автоматической сегментации талька, классификации сульфидных срастаний и вынесения геологического заключения по сорту руды (Рядовая, Труднообогатимая, Оталькованная).

## 1. Архитектура и методы решения

### AI-сегментация талька (Deep Learning)
Для локализации зон оталькования используется глубокая сверточная нейросеть U-Net с бэкбоном ResNet-34. Модель обучена на выборке аннотированных изображений, расширенной с помощью аугментации данных (повороты, отражения, изменения яркости и контраста). Для обработки панорам сверхвысокого разрешения применяется алгоритм скользящего окна (тайлинг) с размером окна 512x512 пикселей и шагом 128 пикселей. Наложение тайлов сглаживается с помощью взвешенной карты вероятностей для исключения видимых границ склейки.

### Выделение сульфидов (Пороговая сегментация)
Сульфидные минералы обладают высоким коэффициентом отражения. Для их точного выделения исходное изображение переводится в цветовое пространство HSV, после чего применяется пороговая бинаризация по каналу яркости (Value > 150). Шум и мелкие артефакты полировки отсекаются с помощью морфологического открытия и удаления контуров площадью менее 100 пикселей.

### Морфологическая классификация срастаний сульфидов (Computer Vision)
Разделение сульфидов на обычные (крупные) и тонкие (прожилковые) выполняется с помощью алгоритма эрозионной реконструкции:
* Вычисляется размер ядра эрозии, динамически адаптирующийся под разрешение кадра (2.5% от максимальной стороны изображения).
* К бинарной маске сульфидов применяется операция эрозии. Мелкие прожилки и осколки полностью стираются, а ядра крупных монолитных зерен выживают.
* Для каждого контура сульфида проверяется пересечение с выжившими ядрами. Если пересечение есть, то все зерно классифицируется как обычное срастание. Если пересечения нет (зерно полностью стерлось при эрозии), оно классифицируется как тонкое срастание.
* Вычисляется доля тонких срастаний среди всех сульфидов в процентах.

### Экспертная логика классификации руды
Сорт руды определяется по трехэтапному алгоритму:
* Если доля площади талька превышает 10.0%, руда классифицируется как оталькованная.
* Если доля площади талька не превышает 10.0%, выполняется классификация по типу сульфидных срастаний. При доле тонких срастаний более 40.0% руда классифицируется как труднообогатимая. При доле тонких срастаний 40.0% и менее руда классифицируется как рядовая.

## 2. Итоговые метрики качества

### Точность сегментации талька
* IoU (Jaccard Index): 0.51
* F1-Score (Dice Coefficient): 0.81
* Относительная погрешность оценки доли талька: не более 3% (соответствует требованиям технического задания).

### Точность классификации сульфидов
* F1-Score классификации руды (Рядовая / Труднообогатимая): 82.96% (метрика рассчитана на полной выборке из 136 тестовых изображений).

### Производительность
* Время обработки панорамного изображения: 7.6 секунд на GPU (RTX 3060 Ti) и до 35 секунд на CPU. Требование технического задания (не более 5 минут) выполнено с многократным запасом.

## 3. Инструкция по запуску

### Docker (рекомендуется)

#### Требования
- [Docker](https://docs.docker.com/get-docker/) 20.10+
- [Docker Compose](https://docs.docker.com/compose/install/) v2+
- Файл весов модели `best_unet_model.pth` в корне проекта

#### Запуск (CPU)
```bash
docker compose up app
```

#### Запуск (GPU, требует [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html))
```bash
docker compose up gpu
```

Приложение будет доступно по адресу: http://localhost:8501

#### Остановка
```bash
docker compose down
```

---

### Установка зависимостей
Убедитесь, что у вас установлен Python версии 3.10 или выше. Установите необходимые библиотеки:
```bash
pip install -r requirements.txt
```

### Запуск веб-приложения
Запустите сервер Streamlit из корневой директории проекта:
```bash
python -m streamlit run app.py
```
После запуска интерфейс будет доступен в браузере по адресу: http://localhost:8501

### Запуск пайплайна обучения и валидации
Для полного воспроизведения процесса подготовки данных, обучения и валидации модели выполните следующие шаги:

1. Импорт разметки из файлов CVAT XML и генерация масок:
```bash
python parse_annotations.py
```

2. Генерация аугментированного набора данных (420 обучающих пар):
```bash
python augmen.py
```

3. Обучение модели U-Net ResNet-34 (40 эпох, веса сохраняются в best_unet_model.pth):
```bash
python train.py
```

4. Валидация качества сегментации талька на размеченных снимках (IoU, Dice, Hausdorff):
```bash
python evaluate.py
```

5. Валидация точности классификации руд на 136 снимках:
```bash
python evaluate_classification.py
```

6. Автоматическая калибровка геометрических порогов классификации сульфидов:
```bash
python calibrate.py
```

## 4. Структура проекта
* app.py: Веб-интерфейс приложения на Streamlit и генератор PDF-отчетов.
* detector.py: Основной пайплайн детекции талька, сульфидов и классификации руды.
* train.py: Скрипт обучения нейросети U-Net.
* augmen.py: Скрипт генерации аугментированного набора данных.
* parse_annotations.py: Парсер аннотаций CVAT XML в бинарные маски.
* evaluate.py: Скрипт валидации качества сегментации талька.
* evaluate_classification.py: Быстрая валидация точности классификации руд.
* calibrate.py: Скрипт подбора оптимальных геометрических порогов эрозии.
* requirements.txt: Список используемых зависимостей.
* Dockerfile: Конфигурация контейнера приложения.
* docker-compose.yml: Конфигурация сборки (CPU и GPU варианты).
* .dockerignore: Файлы, исключённые из сборки Docker-образа.
